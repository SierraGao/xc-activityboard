# 设计规范

> 版本：V1.0
> 更新日期：2026-06-14

---

## 一、设计原则

- **极简风**：减少装饰元素，提高查询效率
- **信息优先**：突出活动信息，弱化UI装饰
- **操作简单**：一次点击可达核心功能
- **移动端优先**：以手机屏幕为基准设计

---

## 二、色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色调 | `#E91E63` | 强调色，用于按钮、标题 |
| 浅色背景 | `#FFF0F3` | 页面全局背景 |
| 主色浅底 | `#FCE4EC` | 卡片 hover 等场景 |
| 白色 | `#FFFFFF` | 卡片、内容区背景 |
| 正文色 | `#333333` | 主要文字 |
| 次要文字 | `#666666` | 标签、辅助信息 |
| 弱化文字 | `#999999` | 时间、提示 |

| 状态 | 色值 | 文字色 | 浅底背景 |
|------|------|--------|---------|
| 进行中 | `#4CAF50` | 绿色 | `#E8F5E9` |

## 三、组件规范

### 3.1 活动卡片

```
┌──────────────────────────────────┐
│ 活动名称              [状态标签]   │
│ YYYY.MM.DD截止                   │
└──────────────────────────────────┘
```

- 圆角：12px
- 阴影：`0 2px 8px rgba(0,0,0,0.08)`
- 内边距：16px
- 点击态：缩放至 98%
- Hover态：上移 1px + 阴影加深

### 3.2 搜索框

- 宽度100%、圆角24px
- Placeholder：「搜索活动名称...」
- 聚焦时边框变为主色调

### 3.3 状态筛选标签

- 横向排列，可滚动
- 激活态：主色调填充 + 白色文字
- 默认态：白底 + 灰色边框 + 灰色文字

### 3.4 详情区块

- 白色背景卡片 + 圆角12px + 阴影
- 每个字段独立一行
- 字段标签在上（灰色小字），内容在下

## 四、字体规范

| 层级 | 大小 | 粗细 | 用途 |
|------|------|------|------|
| H1 | 1.25rem (约20px) | 600 | 页面标题 |
| H2 | 1.2rem (约19px) | 600 | 详情/区块标题 |
| Body | 0.95rem (约15px) | 400 | 正文内容 |
| Caption | 0.85rem (约14px) | 400 | 辅助信息 |
| Small | 0.8rem (约13px) | 400 | 时间、标签 |
| Tiny | 0.75rem (约12px) | 500 | 状态标签 |

字体栈：`-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", Arial, sans-serif`

---

## 五、布局规范

- 最大宽度：480px（手机屏幕宽度）
- 水平居中
- 左右内边距：16px
- 卡片间距：12px
- 区块间距：12px

---

## 六、交互规范

- 卡片点击 → 进入活动详情页
- 搜索框输入 → 实时过滤活动列表
- 状态标签点击 → 切换筛选，默认「进行中」
- 详情页 → 返回箭头回到首页
- 删除操作 → 弹出确认

## 一、项目背景

微信群内活动信息主要通过群消息传播，但由于消息流更新速度快，活动信息容易被刷屏覆盖，导致：
- 用户无法快速找到活动规则
- 新入群用户不了解当前活动
- 用户频繁询问活动参与方式、兑奖方式等
- 运营人员重复答疑成本高

因此建设统一的活动信息查询平台，为社群用户提供稳定、可搜索、可查询的活动公告入口。

---

## 二、产品定位

**名称**：抖B社群活动公告栏

**定位**：微信群活动信息统一查询平台

**目标**：
1. 统一管理活动信息
2. 提供活动搜索能力
3. 降低重复咨询
4. 为后续AI机器人检索提供知识来源

---

## 三、目标用户

| 角色 | 主要行为 |
|------|---------|
| 普通用户 | 查看当前活动、查询活动规则、查询兑奖方式、查询活动入口 |
| 运营人员 | 创建活动、编辑活动、删除活动、管理历史活动 |

## 四、功能需求

### 4.1 前台功能（V1）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 活动列表展示 | P0 | 卡片形式展示所有活动 |
| 搜索活动 | P0 | 按活动名称模糊搜索 |
| 状态筛选 | P0 | 按进行中/待开始/可兑奖/已结束/全部筛选 |
| 活动详情页 | P0 | 展示活动完整信息 |

### 4.2 后台功能（V1）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 新增活动 | P0 | 创建新活动，含基础字段和内容字段 |
| 编辑活动 | P0 | 修改已有活动信息 |
| 删除活动 | P0 | 删除活动记录 |
| 搜索活动 | P0 | 按活动名称搜索 |
| 状态筛选 | P0 | 按状态筛选活动列表 |

### 4.3 暂不开发（V1）

- ❌ 用户系统 / 登录注册
- ❌ 评论互动
- ❌ 收藏 / 点赞
- ❌ AI聊天
- ❌ 消息通知
- ❌ 积分同步
- ❌ 用户中心

## 五、活动状态计算规则

| 状态 | 条件 |
|------|------|
| 待开始 | 当前时间 < 活动开始时间 |
| 进行中 | 活动开始时间 ≤ 当前时间 ≤ 活动结束时间 |
| 可兑奖 | 兑奖开始时间 ≤ 当前时间 ≤ 兑奖结束时间 |
| 已结束 | 当前时间 > 兑奖结束时间 |

**长期活动**：填写开始时间、不填结束时间，系统默认状态为「进行中」，首页显示「长期活动」。

**历史活动**：所有活动永久保留，不自动删除，允许存在多个同名历史活动。
